Automation Rules

The host should not review every message. Rules decide when HostAI can send, when it should wait for approval, and when it must escalate.

Trigger

What happened: guest message, arrival reminder, checkout follow-up, or vendor request.

Guest messageCalendar eventVendor issue

Condition

When the rule should apply: routine intent, confidence threshold, property scope, or channel-specific behavior.

RoutineLow confidenceProperty scope

Action

What HostAI should do: auto-send, draft only, escalate, schedule, or create a maintenance ticket.

Auto-sendDraftEscalate

Rule builder

This is the UI shell for a future rule engine. It is designed so the backend can wire the same concepts later without changing the host-facing experience.

Sample rules

Reply to Wi-Fi requests automatically. Escalate refunds and complaints. Create maintenance tasks from "AC is broken" or "leak" messages.

Default policy

Prefer automation for routine asks, draft for ambiguity, and escalation for anything involving money, safety, or property damage.

Fallbacks

If a PMS is not available, use CSV or manual booking intake to keep the rule engine fed with stay context.
