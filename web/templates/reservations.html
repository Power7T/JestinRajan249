Manual booking fallback

Guest context mapping

Add the guest phone number and room / unit / property number to a reservation. When the guest messages from that number, HostAI matches the reservation, injects the room context into the AI prompt, and replies more naturally.

Examples:
• "Unit 12A" for a suite
• "Room 204" for a hotel room
• "+91 98765 43210" for the guest phone

Arrival activation workflow

The cleanest host flow is simple: booking imported, guest phone saved, room or unit mapped, then the bot can answer in the context of the stay. That makes guest chat feel personal instead of generic.

Arrival ready CSV fallback Room-aware chat Manual booking fallback
1

Import the stay

Use CSV if the host has no PMS. If a PMS exists, keep it as the primary source of truth and CSV as backup.

2

Map guest phone and room

Save the guest phone plus room, unit, or property number. That lets HostAI match the right reservation when the guest writes in.

3

Activate the guest channel

Forward the guest's messages into HostAI. The system should answer using the stay context, house rules, and check-in details.

4

Review exceptions only

Complaints, refunds, maintenance, and missing booking context should be the only things that need deliberate host attention.

What guests can ask

Wi-Fi, parking, towels, early check-in, late checkout, local recommendations, lost keys, and maintenance requests are all fair game for the bot.
